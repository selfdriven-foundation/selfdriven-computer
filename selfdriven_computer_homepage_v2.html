Core idea

From instructions
to proof.

Traditional computing optimised for performance. Agentic computing also needs trust. When software can plan, decide, communicate, and act over long periods of time, the question is no longer just whether it runs. The question is whether it can be understood, constrained, and verified.

selfdriven.computer proposes a clocked, reduced, and inspectable substrate for agent execution. A minimal set of instructions makes behaviour easier to reason about, easier to govern, and easier to connect into a wider trust ecosystem built on KERI/ACDC cryptographic identity.

Clocked trust

A fundamental cycle gives the system rhythm. Every instruction, decision, and state transition anchors to a verifiable moment in time via KERI key events.

Reduced instruction set

Keep the core small. Fewer primitives. Clear semantics. Easier auditability. Better safety. Stronger composability across agents and systems.

Verifiable by design

The operating model is built so actions can be checked, replayed, and reasoned about. Trust comes from cryptographic proofs, not promises.

Ecosystem native

Not just one agent. A coordinated environment of identities, skills, policies, memory, and execution patterns working together at scale.

Architecture

A layered model for trustworthy agent systems.

Each layer narrows ambiguity and increases inspectability. The lower the layer, the smaller and more deterministic the primitive. The higher the layer, the richer the behaviour. Verification travels upward. Complexity is built on top of simple foundations.

Click a layer to explore its role in the stack.

Clock cycle
The fundamental heartbeat. Every instruction and state transition anchors to a verifiable moment. KERI key event sequence numbers provide the universal clock.

02
Instruction primitives
A reduced set of atomic operations — perceive, reason, decide, act, attest. Each primitive maps to a verifiable event type in the KERI event log.
▾
03
Policy and verification
ACDC credentials assert what an agent is allowed to do, for how long, and under what conditions. Policy is cryptographically bound, not configured at runtime.
▾
04
Agent runtime
The execution environment for composable agent skills. Every runtime instance holds a KERI-delegated AID scoped to its operational domain and time window.
▾
05
Shared trust layer
Cross-agent identity resolution, witness networks, and key event log replication. Agents discover and verify each other without a central directory.

Ecosystem applications
Domain services — health, finance, community, professional — built on top of the verified agent substrate. Each inherits the full trust stack from below.

Why it matters

Long-running agents need
stronger foundations.

As AI agents take on longer, more consequential tasks across health, finance, governance, and community — the trust infrastructure they operate on becomes as important as the models themselves. selfdriven.computer is that infrastructure layer.

Verifiable AI agent operating system Every action is signed, sequenced, and replayable. No action without a proof.
Deterministic and inspectable instruction flow Reduced primitives make agent behaviour easier to reason about and govern.
Composable agent skills and protocols SKILL.MD-style capabilities combine safely within a shared trust substrate.
Identity-first trust across agents and humans KERI AIDs and ACDC credentials provide self-sovereign identity at every layer.
Safer long-running automation Time-limited delegated AIDs ensure agents operate within authorised scope.
Evidence trails for governance and compliance Tamper-evident key event logs provide audit-ready records of all agent actions.
Position

Not just AI that works. AI that can be trusted to work within bounds.

selfdriven.computer sits inside a wider ecosystem concerned with identity, trust, governance, and human agency. The goal is not autonomy without limits. The goal is useful agency with verifiable constraints.
